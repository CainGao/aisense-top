📝 小说续写详细流程

续写判断逻辑

锋子每日扫描小说时，判断逻辑：

IF 小说状态 == "ongoing" THEN
    IF 今日 < 计划结束日期 THEN
        生成新章节
        更新章节数
    ELSE
        标记为"completed"
        发送完成通知
    END IF
END IF

每本小说配置：
{
    "novel_id": "xxx",
    "status": "ongoing",
    "update_frequency": "daily",
    "chapters_per_day": 3,
    "start_date": "2026-03-01",
    "end_date": "2026-06-30",
    "total_chapters_planned": 300
}

续写内容质量保证

  • ✅ 上下文连贯：锋子会读取前5章内容，保持剧情连贯
  • ✅ 人物一致：自动维护人物设定和性格
  • ✅ 风格统一：保持小说的整体风格和基调
  • ✅ 情节推进：根据大纲推进剧情发展
  • ✅ 质量检查：自动检测逻辑错误和重复内容

📊 数据库设计

核心表结构

1. novels（小说表）

CREATE TABLE novels (
    id UUID PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    title_en VARCHAR(255),
    author VARCHAR(100) DEFAULT 'AI',
    category VARCHAR(50),
    tags TEXT[],
    description TEXT,
    description_en TEXT,
    cover_url TEXT,
    status VARCHAR(20) DEFAULT 'ongoing',
    total_chapters INTEGER DEFAULT 0,
    free_chapters INTEGER DEFAULT 20,
    price_per_chapter DECIMAL(10,2) DEFAULT 0.05,
    subscription_price DECIMAL(10,2) DEFAULT 9.99,
    view_count INTEGER DEFAULT 0,
    read_count INTEGER DEFAULT 0,
    update_frequency VARCHAR(20) DEFAULT 'daily',
    chapters_per_day INTEGER DEFAULT 3,
    start_date TIMESTAMP,
    end_date TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

2. chapters（章节表）

CREATE TABLE chapters (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    chapter_number INTEGER NOT NULL,
    title VARCHAR(255),
    title_en VARCHAR(255),
    content TEXT NOT NULL,
    content_en TEXT,
    word_count INTEGER,
    is_free BOOLEAN DEFAULT false,
    price DECIMAL(10,2) DEFAULT 0.05,
    view_count INTEGER DEFAULT 0,
    generation_status VARCHAR(20) DEFAULT 'pending',
    translation_status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP DEFAULT NOW()
);

3. generation_tasks（生成任务表）

CREATE TABLE generation_tasks (
    id UUID PRIMARY KEY,
    novel_id UUID REFERENCES novels(id),
    task_type VARCHAR(50),
    status VARCHAR(20) DEFAULT 'pending',
    config JSONB,
    result JSONB,
    error_message TEXT,
    retry_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);

4. cron_logs（定时任务日志表）

CREATE TABLE cron_logs (
    id UUID PRIMARY KEY,
    task_name VARCHAR(100),
    execution_time TIMESTAMP,
    status VARCHAR(20),
    novels_processed INTEGER,
    chapters_generated INTEGER,
    chapters_translated INTEGER,
    errors TEXT[],
    report TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
